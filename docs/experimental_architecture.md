# 🛰️ 多自主平台协同语义引导重识别：实验架构与逻辑指南 (Experimental Architecture & Logic Guide)

为了让团队成员快速理解本系统的实验设计、核心物理逻辑与跨平台数据流，本指南详细梳理了系统的完整技术底座与消歧匹配原理。

---

## 📌 1. 核心协同架构：侦察-支援者（Observer-Supporter）模型

系统严格遵循多自主平台协同制导与重识别（Re-ID）战术任务流设计：

```
 前出侦察平台 (Observer)                精确打击/支援平台 (Supporter)
   [ 视角 image-01 ]                         [ 视角 image-02...10 ]
          │                                         │
          ▼                                         ▼
1. 提取目标属性与地标拓扑关系                  1. 计算真目标像素中心
2. 封装成高层语义目标包 (STP)                   2. 注入高斯空间噪差 (Sigma)
          │                                         │
          ▼                                         ▼
   生成 STP (JSON 格式) ──► 注入噪声坐标 ──► 生成 noisy_point (坐标指针)
          │                                         │
          └─────────────────┬───────────────────────┘
                            ▼
                协同重识别算法决策中心 (Re-ID)
           [ 对照组：坐标法 / 仿真法 / YOLO / VLM ]
```

### 🛰️ 角色职责对齐
* **观察者 (Observer, 如 A-01.json)**：
  前出近距离侦察平台。在高保真视角下，对战场主目标进行细粒度标注，生成**语义目标包 (STP, Semantic Target Package)**。STP 包含：
  - **Category / Label**：目标实体类别（如 `tank`）。
  - **Attributes**：目标视觉细粒度属性特征（如 `camo` 迷彩, `tracks` 履带, `active` 运转状态）。
  - **Relations**：目标与周围地标的拓扑位置关系约束（如 `["tank_1", "near", "hangar"]`）。
  
* **支援被引导者 (Supporter, 如 A-02.json ~ A-10.json)**：
  精确打击或支援平台，处于远距离或电磁对抗环境。
  - 系统获取 Supporter 视角中真目标的像素中心坐标，动态注入高斯噪声 $\sigma$ 偏移，得到**降级空间坐标指针 `noisy_point`**。
  - Supporter 平台仅依赖此退化的 `noisy_point` 指针和 Observer 传回的 STP 语义包，在存在相近 decoy（诱饵）的强混淆环境下，执行 Re-ID 决策消歧。

## 📌 2. 四种重识别对照算法逻辑

系统集成了四种不同维度的算法路线，用以横向对比抗干扰鲁棒性与性能边界：

### 📍 路线 A：纯位置坐标匹配 (Coordinate Baseline)
* **决策逻辑**：最邻近搜索（Nearest Neighbor）。
* **数学表示**：
  $$\text{Selected} = \arg\min_{c \in C} \text{Distance}(\text{Center}(c), \text{noisy\_point})$$
* **特点与漏洞**：纯几何方法。当噪差 $\sigma$ 增大，或者真目标旁布设了同类别干扰物（decoy）时，该方法会产生毁灭性的误锁（Mislock），极易被引流锁定到诱饵上。

### 📐 路线 B：语义属性理论仿真 (Semantic Simulation)
* **决策逻辑**：五维数学评分矩阵消歧。
* **数学评分公式**：
  $$\text{Score} = w_{coord} \cdot S_{coord} + w_{cat} \cdot S_{cat} + w_{attr} \cdot S_{attr} + w_{rel} \cdot S_{rel} + w_{scene} \cdot S_{scene}$$
  各维度权重可通过 `experiment_config.yaml` 灵活微调：
  - `coordinate` ($S_{coord}$): 候选物中心到 noisy_point 的欧氏距离衰减评分。
  - `category` ($S_{cat}$): 实体类别是否 100% 匹配。
  - `attribute` ($S_{attr}$): 细粒度视觉属性重合度评分（Jaccard 相似度）。
  - `relation` ($S_{rel}$): 空间拓扑关系几何符合度校验（Left_of, Right_of, Near 等几何计算）。
  - `scene` ($S_{scene}$): 宏观场景标签上下文匹配度。
* **特点**：在 CPU 内存中进行毫秒级数学仿真计算，是论文算法的理论性能上限（Performance Ceiling）。

### 🧠 路线 C：大模型在线物理闭环 (Grounded VLM)
* **决策逻辑**：全图指代注意力机制（Grounded Full-Image Attention）+ CoT 推理。
* **核心流向**：
  1. **全图无损加载**：将支援平台图像无损转换为 Base64 喂给大模型，避免裁切造成空间关系丢失。
  2. **双坐标参考系注入**：在 Prompt 中同时注入**绝对像素坐标**与 **$[0-1000]$ 归一化坐标**，引导本地 Qwen2-VL-7B 的注意力聚焦到特定候选实体。
  3. **自愈感知**：系统启动时自动向推理机拉取 `/v1/models` 接口，自动握手对齐当前显卡上承载的量化大模型，零配置热插拔。
  4. **CoT 论证决策**：大模型执行思维链（CoT）分步推理（实体视觉属性比对 $\to$ 空间拓扑地标比对），最终在末行输出 `最终判定：是/否`。

### 🔮 路线 D：YOLOv8 传统检测基线 (YOLO Baseline)
* **决策逻辑**：通用检测器 + 置信度最高优先规则（Highest Confidence Selection）。
* **核心流向**：
  1. 调用 `yolov8n.pt` 对支援平台图像运行全图检测。
  2. **类别重映射**：自动将 COCO 预训练类别中的 `boat` (8)、`truck` (7)、`car` (2) 等重映射为 `"tank"` 候选框。
  3. **决策选择**：遇到多目标时，**直接选择置信度最高的检测框作为匹配目标**。
* **学术价值**：证明了非语义检测算法由于缺乏拓扑逻辑理解，存在永久性的**精度天花板（本实验中恒定为 47.83%）**，一旦 decoy 的置信度高于真目标，检测器将 100% 被诱骗锁定。

## 📌 3. 强干扰战场环境（Decoy 诱饵）模拟机制

为了在标注数据的基础上，科学检验语义消歧的必要性，系统在运行实验时，会自动执行**同类别相近 Decoy 干扰注入**：

```
                 [ dx = 120px, dy = 60px ]
  ┌───────────────────┐               ┌───────────────────┐
  │                   │               │                   │
  │    真目标 (Tank)   ├──────────────►│   Decoy 诱饵干扰   │
  │     [tank_1]      │               │     [tank_2]      │
  │                   │               │                   │
  └───────────────────┘               └───────────────────┘
   (符合 STP 属性与地标拓扑)             (相同 Category 类别)
                                      (不同 Attributes/Relations)
```

* **逻辑实现**：
  在 `semantic_guidance/experiment.py` 的单次实验调度器中，如果场景内没有标注 distractor（干扰物），系统会自动在真目标坐标中心偏移 `[dx=120, dy=60]` 像素处，**动态生成一个相同品类、相同大小的 decoy（诱饵）**。
* **考量指标**：
  - **Success Rate（匹配成功率）**：正确锁定真目标（`target_object_id`）的比例。
  - **Mislock Rate（误锁率 / 虚警锁定率）**：由于噪差坐标偏移或置信度诱骗，导致系统锁定到 decoy 诱饵上的比例。

## 📌 4. 跨平台网络拓扑与直连协议

为了在开发协同中保障服务稳定性，系统架构划分为 **Web 控制端** 与 **量化推理端** 两部分，完全采用**局域网物理直连**：

```
  [ 我的 Windows 电脑 (本地工作区) ]                 [ 2080Ti Linux 推理机 ]
        IP: 192.168.2.xxx                              IP: 192.168.2.63
 ┌──────────────────────────────┐              ┌───────────────────────────────┐
 │  Flask 后端服务 (app.py)     │              │  vLLM 推理服务 (Port: 8000)   │
 │   - Web 前端 (HTML/CSS/JS)   │              │   - Qwen2-VL-7B-Instruct-AWQ  │
 │   - 实验调度与数据聚合       │              │   - 绑定监听 0.0.0.0          │
 └──────────────┬───────────────┘              └───────────────▲───────────────┘
                │                                              │
                └─────────────── HTTP REST API ────────────────┘
                            (直接通信，无需 SSH 隧道)
```

### ⚙️ 推理端核心配置规范
为了在 **11G 显存** 的 2080Ti 上顺畅加载 7B 大模型，推理端需严格遵守以下参数规范（已固化在 `deploy_vllm_7b.sh` 中）：
* **`--quantization awq`**：4-bit 硬件级压缩，降低底座显存至 3.5GB。
* **`--enforce-eager`**：禁用 CUDA Graph，腾出 **1.48GB** 显存。
* **`--max-model-len 4096`**：截断上下文长度，完美容纳大图视觉 Token。
* **`--host 0.0.0.0`**：向局域网暴露，允许 Windows 机器直连访问。
